### Introduce
소스코드 작성 전에 반드시 블록 다이어그램을 통해 먼저 설계하는 저만의 원칙을 지키므로 발생 가능한 문제들을 사전에 대비하기 위해 노력하고 있습니다.  
코드 작성 이후에는, 지속적인 리팩토링과 테스터 작성을 통해 안전성을 높이는 데 힘쓰고 있습니다.

"Done is better than perfect". 빠르게 기술 부채를 갚아 나가는 개발자가 되고 싶습니다.
